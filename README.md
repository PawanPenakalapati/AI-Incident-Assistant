# AI Incident Analysis Assistant

## Overview
This project demonstrates an AI-driven workflow to analyze incident data and generate structured outputs for operational and leadership decision-making.
## Workflow
![Workflow](workflow.png)
## Problem
Incident data is often unstructured, inconsistent, and spread across multiple teams, making it difficult to identify root causes, define actions, and ensure accountability.

## Solution
This tool uses a structured input format and a controlled AI prompt to transform incident data into a standardized report including:
- Incident Summary  
- Business Impact  
- Observations  
- Corrective Actions  
- Preventive Actions  
- RACI Matrix  
- Root Cause Note  

## Input
The system expects structured incident data with fields such as:
- Incident Title  
- Reported Time  
- Systems Affected  
- Symptoms Observed  
- Actions Taken  
- Additional Notes / Logs  
- Repeat Issue  

## Output
A clear and actionable report that:
- Highlights key issues and impact  
- Identifies gaps and risks  
- Provides prioritized corrective and preventive actions  
- Assigns ownership using RACI  

## Value
- Reduces manual effort in incident reporting  
- Improves consistency and quality of RCA  
- Enables faster and better decision-making  
- Enhances accountability across teams  

## How it works
Structured Input → AI Prompt → Standardized Output

## Use Case
Designed for IT operations, service delivery teams, and program leadership to streamline incident analysis and governance.
